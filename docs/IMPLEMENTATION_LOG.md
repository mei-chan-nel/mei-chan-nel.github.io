# 学習サイト改修・リポジトリ分離記録

作業日: 2026-07-12、2026-07-13、2026-07-14

## リポジトリ境界

公開サイトは独立した2リポジトリで管理する。

### `mei-chan-nel.github.io`

- ポータルトップ
- このサイトについて
- 全体プライバシーポリシー
- ポータル共通デザイン
- ホスト直下の `ads.txt` と `robots.txt`
- ポータル、問題一覧、アプリをまとめた全体 `sitemap.xml`
- フッターから閲覧できる利用者向け `sitemap.html`
- AdSense再審査設定と全体評価

### `info1-quiz-app`

- 既存学習アプリ `app/`
- 問題データと分野分類
- 問題一覧トップと6分野の生成ページ
- 問題一覧用デザイン
- 問題一覧生成・分類・検証
- ポータルのサイトマップ生成へ渡す問題一覧ビルド記録

## 変更禁止範囲

既存学習アプリの出題・遷移・集計処理は変更しない。今回の作業開始時にGit差分がなかった `app/index.html`、`app.js`、`startup.js`、`styles.css`、`issue-report.js`、`issue-report.css` のSHA-256を `info1-quiz-app/docs/reports/app-core-baseline-sha256.json` に保存し、問題一覧検証時に一致を確認する。

## 第1段階: 問題分類

`info1-quiz-app` で実施した。

- 完成済みの全1,000問へ主分野を1件設定した。
- 自動候補と一致する889問、初期分類で個別確認した69問、完成済みデータの主分野を保持した42問を記録した。
- `question.schema.json` で `field_ids` を新規問題の必須項目とした。
- 未分類は0問。

分類結果:

- 社会・セキュリティ: 359問
- デジタル表現: 216問
- ネットワーク: 150問
- データ活用・DB: 113問
- アルゴリズム: 99問
- 情報デザイン: 63問

## 第2段階: 問題一覧

`info1-quiz-app` で実施した。

- 問題一覧トップと6分野・102ページを生成した。
- 1ページ最大10問とした。
- 1,000問を重複なしで1回ずつ掲載した。
- 問題文、選択肢、正答、解説、出典、タグを掲載した。
- `scripts/generate_question_pages.py` は `questions/` とプロジェクトサイトマップだけを生成する。
- `scripts/validate_question_pages.py` は問題一覧、アプリ保護ハッシュ、リポジトリ境界を検証する。

## 第3段階: ポータル

`mei-chan-nel.github.io` で実施した。

- ポータルトップを学習コンテンツ中心の構成へ刷新した。
- 6分野の紹介、学習方法、問題作成方針、問題一覧・アプリへの導線を追加した。
- 完成済み問題に合わせ、トップの件数を1,000問・6分野・重複除外613タグへ更新した。
- サイト案内とプライバシーポリシーを整備した。
- `assets/site.css` と `assets/favicon.svg` をポータルだけの共通資産とし、問題一覧から同じ公開URLを参照する。
- 旧 `styles.css` は参照されなくなったため削除した。

## 第3.5段階: 動画問題と書籍案内

`mei-chan-nel.github.io` で2026年7月13日に実施した。

- 原本Excelの5分野・330問から、問題・答え・動画番号だけを公開用JSONへ抽出した。品質が一定でない原本のキーワード列は使用しない。
- Amazon KDPの契約対象となる解説本文は、公開用JSONにもHTMLにも収録しない。
- YouTubeチャンネルの公開動画を番号で照合し、330問へ333本を対応付けた。番号231・311・312は各2本である。
- 動画問題トップと5分野・35ページを1ページ10問で生成した。
- YouTube埋め込みはプライバシー強化モードを使い、利用者がボタンを押したときだけ読み込む。
- Kindle問題集4冊の案内ページを追加し、価格など変動する情報はAmazon側で確認する構成とした。

## 2026年7月14日: トップ導線とページ分割の再設計

- 学習アプリへのCTAをトップの6分野より上へ移し、領域全体をリンクにした。
- トップの6分野を「アプリ掲載問題」と明示した。
- 動画問題を「情報社会・デザイン」「デジタル」「ネットワーク」「プログラミング」の4分類へ統合し、トップへ説明付きカードを設置した。
- 動画問題とアプリ掲載問題の双方を1ページ最大10問に統一した。
- Kindle問題集4冊の書影をローカル資産として保存し、トップと書籍案内ページへ掲載した。
- 利用者向け `sitemap.html` を新設し、ポータルと生成問題ページの全フッターからリンクした。

## 2026年7月14日: 動画問題の可読性とキーワード監査

- Q001〜Q330を問題文・正答・分類・対応動画題名から1問ずつ見直し、既存値を参照せずキーワードを再構築した。
- 全330問へ2〜4個、延べ955個・重複除外618語を割り当てた。割り当て根拠と公開データのSHA-256は `docs/video-keyword-audit.json` に記録する。
- キーワード再構築を独立した生成工程にし、原本Excelを再取込しても古いキーワード列が混入しないようにした。
- 問題文は文意に合わせて改行し、プログラミング100問のプログラム部分を横スクロール可能・自動折返しなしのコードブロックにした。
- 動画はプライバシー強化モードの遅延埋め込みだけに統一し、「YouTubeで見る」の直接リンクを全問題から削除した。

## 2026年7月14日: 書籍案内と配色の調整

- トップと書籍詳細ページの4冊を、小さな書影と説明が1冊ずつ対応する縦並びへ統一した。
- スマートフォンでは書影幅を抑え、説明欄に可変幅を割り当てて横はみ出しを防いだ。
- トップの標語を「知識を、／ひろげ、／つなげる」の3行へ変更し、各ページのフッター表記も統一した。
- プログラムのコードブロックを、薄く色味を付けた白系背景と黒系文字へ変更した。

## 第4段階: 広告と機械可読ファイル

- ポータルトップ、問題一覧トップ、102分野ページ、動画問題35ページ、既存アプリ本体を広告対象とした。
- ポータルの案内・プライバシー・書籍案内・HTMLサイトマップは広告コードなしとした。
- サイト案内とプライバシーポリシーをポータルへ統合し、アプリのフッターからポータルのトップ・案内・プライバシーへ直接リンクした。
- ホスト単位で参照される `ads.txt` と `robots.txt` は `mei-chan-nel.github.io` だけに置いた。
- 両リポジトリのビルド記録から公開ページ一覧を読み、`mei-chan-nel.github.io/sitemap.xml` に両リポジトリの144 URLを収録する。
