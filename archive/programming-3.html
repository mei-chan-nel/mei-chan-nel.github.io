QUESTION 251

映画館の座席が5行3列の2次元配列Seatsで管理されている。座席は0が空席、1が予約済みを表す。現在のSeatsの要素の値は以下の通りになっている。なお、この配列の添字は1から始まる。 0 1 0 1 1 0 0 1 0 0 0 1 1 0 0 次のプログラムの実行した際の表示結果を答えよ。 （1）gyou = 3 （2）retu = 2 （3）表示する( gyou, "行", retu, "列目の座席の状態は", Seats[gyou, retu], "です。")

答えを見る

答え3行2列目の座席の状態は1です。 （予約済みです）

キーワード
  • このプログラムは2次元配列を用いて映画館の座席の予約状況を管理するものである。 共通テスト用プログラム表記では、特に指定がない場合、行番号と列番号は0から始まるが、今回は添字を1から始めるように指示がある。 つまり、最前列から1行目、2行目、3行目となり、横の位置は左から1列目、2列目、3列目と表される。 添字を1から始めることで日常的な数え方とプログラム上での数え方が一致し、直感的にわかりやすくなる。 このプログラムでは3行2列目の座席の状態を表示している。 1・2行目：変数gyouに値3、変数retuに値2を代入する。 3行目：Seats[3
  • 2]の値は1なので、「3行2列目の座席の状態は1です。」と表示し、プログラムは終了する。

QUESTION 252

次のプログラムの実行した際の表示結果を答えよ。 （1）x = 7 （2）もし x < 10 ならば： （3）｜ 表示する("xに10を加える。") （4）⎿ x = x + 10 （5）表示する("xの値は", x)

答えを見る

答えxに10を加える。 xの値は17

キーワード
  • このプログラムは条件分岐の基本に関するものである。 共通テスト用プログラム表記における条件分岐は「もし【条件】ならば：」という形式で記述される。 【条件】の部分が成立する（真である・True）とき、｜と⎿で囲まれた部分を実行する。 【条件】が成立しない（偽である・False）とき、｜と⎿で囲まれた部分を実行せずに⎿の次の行まで飛ぶ。 ⎿の次の行からは、【条件】が成立する・しないに関わらず、実行される。 以下プログラムの解説。 1行目：変数xに数値7を代入する。 2行目：条件「x < 10」を評価する。今、変数xの値は7なので、「7 < 10」となり、この不等式は正しい（成立する・真である・True）。よって、｜と⎿で囲まれた部分（3・4行目）を実行する。 3行目：「xに10を加える。」と表示する。 4行目：まず右辺を計算結果の17を変数xに代入する。 条件分岐による処理はここまでである。 5行目：この行からは、【条件】が成立する・しないに関わらず実行される。現在のxの値は17なので、「xの値は17」と表示し、プログラムは終了する。

    キーワード
